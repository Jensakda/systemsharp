static

Converts value to its signed fixed-point representation, using intWidth integer bits and fracWidth fractional bits.

If intWidth is less than 64, an overflow might occur. The behavior of this method in the presence of overflows depends on the currently selected overflow mode (

See Also
FixedPointSettings

). If fracWidth is greater than 0, the fractional part is in fact padded with binary zeroes. If fracWidth is less than 0, some lower bits are cut, which might cause a loss of precision during the conversion.

Re-interprets value as a fixed-point-number with fracWidth fractional bits. I.e. the binary two's complement representation of value is re-interpreted. E.g. we get that SFix.FromSigned(Signed.FromLong(3, 3)).DoubleValue == 1.5.

Definition at line 577 of file SFix.cs.

static SFix SystemSharp.DataTypes.SFix.FromSLV ( StdLogicVector  slv,
int  fracWidth 
)
static

Interprets logic vector slv as a fixed-point number with fracWidth fractional bits. The total width matches the length of the logic vector.

Logic values of '1' and 'H' are interpreted as ones, all other values are interpreted as zeroes. Please keep this in mind, since logic vectors containing logic values such as 'Z' or 'X' may lead to unexpected results.
