Detailed Description

A XIL-S instruction

XIL-S instructions presume a stack machine execution model - very much like Java bytecode and CIL. A XIL-S instruction pops 0 or more operands from stack stack and finally pushes 0 or more results onto the stack. Stack elements are not restricted to have any particular size (as opposed to Java VM, where the stack is strictly 32 bit words). In fact, a stack element can represent any data item, from a simple bool to a 10^6 bit wide fixed point number. Therefore it is of utmost importance that any XIL-S instruction clearly indicates the types of operands it expects on the stack and the types of operands it will put on the stack.
