Detailed Description

Represents a synthesis plan.

The synthesis plan represents the actual design flow which is applied to a clocked process (at algorithm level) in order to transform it into hardware. It constitutes the "heart" of HLS, which currently consists of the following steps:

1. Compile the SysDOM representation to XIL-S code (only if necessary, i.e. if there is no XILSFunction attribute).

2. Apply code transformations at XIL-S level.

3. Transform to XIL-3 representation.

4. Apply code transformations at XIL-3 level.

5. Perform scheduling and allocation, based on registered XIL mappers.

6. Perform interconnect allocation.

7. Perform control path construction.

Each of those transformation steps can be configured in various ways - see documentation below.

void SystemSharp.Analysis.M2M.HLSPlan.AddXILMapper ( Type  dpuType)

Adds a XIL mapper which is identified by its type or the type of its functional unit.

Parameters
dpuTypeThe type of the XIL mapper or the type of the functional unit.

If dpuType is the type of a XIL mapper, it will be instantiated by reflection. In this case, make sure that there is a constructor without arguments. If it is the type of a functional unit, its corresponding mapper class is identified by the DeclareXILMapper attribute. In that case, make sure that such attribute is declared.
